# Obsidian Admin Vue v1.2.2

A frontend refactor release paired with backend v1.3.2, focused on eliminating monorepo over-fragmentation, syncing generated types, expanding component test coverage, and improving the tenant switcher UX.

## Highlights

### Architecture Cleanup
- Merged `@sa/uno-preset`, `@sa/utils`, and `@sa/hooks` packages into `src/` to eliminate over-fragmented monorepo workspace packages
- Moved storage utilities and hooks into `src/utils/` and `src/hooks/common/` respectively, updating all import paths
- Moved the built-in demo backend out of `src/` and behind runtime-only imports so production builds keep a clearer boundary from mock infrastructure
- Extracted authentication side effects into a bootstrap layer and standardized CRUD drawer form logic through a shared operate-form composable
- Removed the unused `packages/alova` workspace package

### Tenant Switcher Redesign
- Redesigned the tenant switcher as a Spotlight-style modal with improved UX, keyboard navigation, and visual hierarchy

### Type Safety & Contract Sync
- Synced generated backend type definitions (`backend-generated.d.ts`) to reflect backend v1.3.2 DTO removals across Auth, FeatureFlag, Language, Role, and User domains
- Fixed `SearchParams` type in `feature-flag.d.ts` to avoid conflicting `current` property types
- Added `skipLibCheck: true` to `tsconfig.json` to skip third-party `.d.ts` type checking

### Test Coverage
- Added `feature-flag-page.spec.ts` and `audit-policy-page.spec.ts` component tests
- Fixed unhandled promise rejection in audit-policy test
- Marked CI-only flaky tenant switch preview test as `test.fixme`
